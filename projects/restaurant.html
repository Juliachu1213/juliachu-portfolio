1. Introduction

The EGo Book system was designed as an integrated restaurant management and customer membership platform. It allows consumers to make online reservations, manage their booking history, and accumulate loyalty points through repeat visits. Restaurants, on the other hand, can register their businesses, manage seating arrangements, track reservation histories, and maintain blacklists for no-show customers.

The goal of this project was to develop a scalable and efficient relational database system that supports dynamic restaurant registration, automated seat allocation, membership rewards, and consumer analytics ‚Äî providing both customers and restaurants with a unified digital service.

2. System Requirements

The system fulfills the four fundamental database operations: Create, Read, Update, and Delete (CRUD), supporting two primary user roles:

  • Customer functions: Register and edit member profile, create or cancel reservations, check available tables, review booking history, and view reward points.
  • Restaurant functions: Register restaurants, manage reservations and blacklists, update seat availability, and record customer spending.

4. System Architecture

The system follows a three-tier architecture with clearly separated layers:

  • Frontend: HTML, CSS, JavaScript ‚Äî responsible for the user interface and dynamic content rendering.
  • Backend: Python FastAPI ‚Äî handles API endpoints, authentication, and business logic.
  • Database: MySQL ‚Äî stores structured data for restaurants, users, reservations, and loyalty programs.

5. Implementation & Key Features

  • Membership system: Tracks user spending and updates loyalty points automatically after each completed reservation.
  • Automated seat allocation: Seats are dynamically assigned based on table availability and party size.
  • Blacklist management: Customers with repeated no-shows are automatically flagged and restricted from future bookings.
  • Data integrity: All CRUD operations are transaction-safe using MySQL‚Äôs ACID compliance.

6. Future Work

  • Integrating machine learning models to predict peak booking times and recommend promotional strategies.
  • Implementing role-based access control (RBAC) for finer-grained permissions between customers and restaurant managers.
  • Adding RESTful APIs for mobile app integration and cross-platform synchronization.

7. Summary

The EGo Book project demonstrates the design of a fully functional restaurant reservation and membership management platform, integrating front-end interfaces, RESTful API services, and relational database management. Through structured schema design and modular development, the system successfully meets both customer convenience and restaurant operational needs.

This project is part of a university database systems course, developed primarily in Traditional Chinese as the course was conducted in Chinese. The user interface, documentation, and database schema therefore use Chinese labels, while the system logic and backend follow standard English-based programming conventions.
