Smart Home Gateways Based on W3C Web of Things Architecture

Overview

This thesis addresses the limitations of the traditional CWMP (CPE WAN Management Protocol) used in smart home gateways, which suffers from SOAP overhead, inefficient event notifications, and weak security mechanisms.

By introducing the Web of Things (WoT) architecture, the research redefines CWMP’s SOAP-based RPC methods into RESTful APIs, improving interoperability and system modularity. Furthermore, WebSocket and Webhook are implemented to replace the CWMP Inform mechanism, significantly enhancing real-time event communication.

Objectives

  • Enhance interoperability through RESTful API-based method mapping following WoT standards.
  • Improve event notification performance via WebSocket and Webhook.
  • Strengthen authentication and authorization using OAuth 2.0 and JWT.

System Design & Implementation

The proposed WoT–CWMP architecture integrates multiple key technologies:

  • Thing Description (TD): Defines device and service resources in JSON-LD format for unified access.
  • OAuth 2.0 & JWT: Enables secure user authentication and resource authorization.
  • Database Layer (PostgreSQL): Stores device configurations, client credentials, and event logs.
  • Spring Boot Framework: Implements WoT scripting logic in Java, re-architecting the CWMP workflow.
  • API Testing: Conducted using Postman with GitHub OAuth integration for validation.

Experiment & Results

  • 30% reduction in event notification latency using WebSocket/Webhook.
  • Higher stability under 5,000+ concurrent requests through load testing.
  • Improved maintainability and scalability from modular RESTful interfaces.

Conclusion

This thesis proposes a secure and efficient smart home gateway management system based on the Web of Things framework. By integrating RESTful APIs, OAuth 2.0, and JWT, the system achieves better interoperability, reduced latency, and stronger security, providing a feasible reference model for next-generation IoT gateway architectures.
